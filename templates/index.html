Project Data Collection (Real-World Implementation)

The input parameters used in this system such as bullet velocity, body mass, kinetic energy, temperature rise, blood loss, and ECG readings are assumed to be collected through real-time sensors attached to an injured soldier in the battlefield.

These sensors can be connected to a microcontroller such as Arduino or ESP32. The microcontroller reads the sensor values and transmits the data using wireless communication technologies like WiFi, GSM, or LoRa.

The transmitted sensor data is then sent to this web application through an API or HTTP request. The system processes the data using a Machine Learning model (KNN) to predict the health status of the injured personnel and identify nearby hospitals for emergency evacuation.

Technology Stack Used:
• Sensors (ECG, Temperature, Blood Loss Sensors)
• ESP32 Microcontroller
• Wireless Communication (WiFi)
• Python Flask Web Application
• Machine Learning Model (KNN - Scikit-Learn)
• SQLite Database
• Deployment using Railway Cloud Platform
